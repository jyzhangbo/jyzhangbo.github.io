源代码安装zabbix

发表于 2018-09-12 | 分类于 zabbix

源代码安装zabbix

关闭selinux和iptables

systemctl stop firewalld.service 
setenforce 0

下载源码

https://www.zabbix.com/download_sources

通过上述的地址下载最新的源代码，然后解压。

创建zabbix用户

groupadd zabbix
useradd -g zabbix zabbix

创建数据库

我这里使用的是外部数据库，创建步骤为：

  1. 创建zabbix数据库
  2. 执行目录./database/mysql目录下的sql文件，需要注意的是执行顺序为:schema -> images -> data

编译

只安装了server和agent,如果需要安装其他，请参考官网

./configure --enable-server --enable-agent --with-mysql --enable-ipv6 --with-net-snmp --with-libcurl --with-libxml2

在编译过程中会遇到各种问题，原因为有些依赖没有安装，安装提示安装即可，需要的有myql-dev，libxml2-devel,net-snmp-devel，gcc等

安装

make install

/usr/local/sbin/zabbix_server -V 验证安装成功

修改配置文件

zabbix_server配置文件

路径:/usr/local/etc/zabbix_server.conf
将数据库配置修改为使用的

zabbix_agent配置文件

路径:/usr/local/etc/zabbix_agent.conf
将配置文件中的Server、ServerActive、Hostname参数设置成zabbix服务器的IP地址即可

Server=10.16.44.109
ServerActive=10.16.44.109
Hostname=127.0.0.1

设置开机启动

cp ./misc/init.d/fedora/core/zabbix_*  /etc/init.d/ -v
chkconfig --add zabbix_server
chkconfig --add zabbix_agentd
chkconfig --level 35  zabbix_server on
chkconfig --level 35  zabbix_agentd  on
chkconfig --list

启动server和agent

/etc/init.d/zabbix_server start
/etc/init.d/zabbix_agentd start

遇到的问题:cannot create Semaphore: [28] No space left on device

解决方案：

修改/etc/sysctl.conf
添加如下行：
kernel.sem = 500        64000   64      256
修改之后，执行sysctl -p使其生效。

前端web配置

将源码中的php代码拷贝到apache目录
mkdir /var/www/html/zabbix/
cp -a ./frontends/php/  /var/www/html/zabbix
更改zabbix网页文件权限
chown -R zabbix.zabbix  /var/www/html/zabbix/
重启apache Web服务器
systemctl restart httpd

浏览器登陆进行zabbix安装初始化

访问地址:http://ip/zabbix/

  1. 安装要求修改php配置文件/etc/php.ini
  2. 重启apache服务
  3. 配置数据库连接
  4. 服务器配置
  5. 最后需要下载配置文件，按要求放入指定目录
  6. 登陆，用户名Admin,密码zabbix

修改语言

administration->users->language->Chinese(zh_CN)->update
