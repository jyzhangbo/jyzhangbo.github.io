hbase分布式安装

发表于 2018-05-23

各个安装软件的版本
jdk | jdk-8u171-linux-x64.tar.gz
hadoop | hadoop-2.7.6.tar.gz
hbase | hbase-1.2.6-bin.tar.gz
zookeeper | zookeeper-3.4.10.tar.gz

本次安装使用了三台机器

master | nameNode
slave1 | dataNode
slave2 | dataNode

1.jdk的安装

需要使用Oracle的jdk,将原装的openjdk卸载，具体步骤请自行百度

2.ssh免密登陆

1.在每台机器上执行下面命令,用来生成一对公钥和私钥

[root@localhost ~]# ssh-keygen -t rsa
一直确定即可完成生成，在/root/.ssh目录下就会生成id_rsa和id_rsa.pub两个文件

2.在.ssh目录下创建authorized_keys文件
3.使用命令将其他机器上的公钥内容添加到目标机器的authorizd_keys文件中

cat id_rsa.pub | ssh root@10.16.44.112 'cat >> .ssh//authorized_keys'
或者
ssh-copy-id root@10.16.44.108

4.完成ssh免密登陆后可以使用scp命令将安装包发送到各个机器上

3.修改主机名，配置hosts文件

修改主机名：hostnamectl set-hostname Linuxidc
修改 /etc/hosts文件

4.关闭和禁用防火墙

systemctl status firewalld.service  查看防火墙状态
systemctl stop firewalld.service  关闭防火墙
systemctl disable firewalld.service   禁用防火墙

5.安装hadoop集群

1..解压安装包后，配置环境变量
2.创建三个文件夹
mkdir /data1/hadoop/tmp
mkdir /data1/hadoop/hdfs/data
mkdir /data1/hadoop/hdfs/name
3.修改配置文件，配置文件都在~/etc/hadoop目录下

1.hadoop-env.sh和yarn-env.sh添加

export JAVA_HOME=/data1/soft/jdk1.8.0_171/

2.core-site.xml

<configuration>
    <property>
        <name> fs.default.name </name>
        <value>hdfs://master:9000</value>
        <description>指定HDFS的默认名称</description>
    </property>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://master:9000</value>
        <description>HDFS的URI</description>
    </property>
    <property>    
        <name>hadoop.tmp.dir</name>
        <value>/data1/hadoop/tmp</value>
        <description>节点上本地的hadoop临时文件夹</description>
    </property>
</configuration>

3.hdfs-site.xml

<configuration>

    <!-- 设置namenode的http通讯地址 -->
    <property>
        <name>dfs.namenode.http-address</name>
        <value>10.16.44.108:50070</value>
    </property>

    <!-- 设置namenode存放的路径 -->
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>/data1/hadoop/hdfs/name</value>
    </property>

    <!-- 设置hdfs副本数量 -->
    <property>
        <name>dfs.replication</name>
        <value>2</value>
    </property>
    <!-- 设置datanode存放的路径 -->
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>/data1/hadoop/hdfs/data</value>
    </property>    

</configuration>

4.输入 cp mapred-site.xml.template mapred-site.xml 将mapred-site.xml.template文件复制到当前目录，并重命名为mapred-site.xml

<configuration>

    <!-- 通知框架MR使用YARN -->
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>

</configuration>

5.yarn-site.xml

<configuration>

    <!-- Site specific YARN configuration properties -->

    <!-- 设置 resourcemanager 在哪个节点-->
    <property>
        <name>yarn.resourcemanager.hostname</name>
        <value>master</value>
    </property>

    <!-- reducer取数据的方式是mapreduce_shuffle -->
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>

    <property>
         <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
         <value>org.apache.hadoop.mapred.ShuffleHandler</value>
    </property>

</configuration>

6.slaves

slave1
slave2

7.将hadoop通过scp命令复制到其他机器上

8.在master机器上执行命令格式化

hdfs namenode -format

9.启动，在sbin目录下执行

./start-all.sh

到此hadoop已经安装完毕，可以访问master的ip+50070端口和8088端口，查看

6.安装zookeeper

1.修改配置文件

# The number of milliseconds of each tick
tickTime=2000

# The number of ticks that the initial 
# synchronization phase can take
initLimit=10

# The number of ticks that can pass between 
# sending a request and getting an acknowledgement
syncLimit=5

# the directory where the snapshot is stored.
# do not use /tmp for storage, /tmp here is just 
# example sakes.
dataDir=/data1/hadoop/zookeeper

# the port at which the clients will connect
clientPort=2181

# the maximum number of client connections.
# increase this if you need to handle more clients
#maxClientCnxns=60
#
# Be sure to read the maintenance section of the 
# administrator guide before turning on autopurge.
#
# http://zookeeper.apache.org/doc/current/zookeeperAdmin.html#sc_maintenance
#
# The number of snapshots to retain in dataDir
#autopurge.snapRetainCount=3
# Purge task interval in hours
# Set to "0" to disable auto purge feature
#autopurge.purgeInterval=1

server.1=master:2888:3888
server.2=slave1:2888:3888
server.3=slave2:2888:3888

2.在dataDir目录下创建文件myid,并写入server后边的数字

3.在各个机器上执行 

./zkServer.sh start

7.安装hbase

1.将hadoop下的core-site.xml和hdfs-site.xml复制到conf目录下
2.修改配置文件hbase-env.sh
export JAVA_HOME=/data1/soft/jdk1.8.0_171
# Tell HBase whether it should manage it's own instance of Zookeeper or not.
export HBASE_MANAGES_ZK=false
3.修改配置文件hbase-site.xml
<configuration>

    <property>
        <name>hbase.rootdir</name>
        <value>hdfs://master:9000/hbase</value>
    </property>
    <property>
        <name>hbase.master</name>
        <value>master</value>
    </property>
    <property>
        <name>hbase.cluster.distributed</name>
        <value>true</value>
    </property>
    <property>
        <name>hbase.zookeeper.property.clientPort</name>
        <value>2181</value>
    </property>
    <property>
        <name>hbase.zookeeper.quorum</name>
        <value>master,slave1,slave2</value>
    </property>    

</configuration>
4.配置regionservers
slave1
slave2
5.启动
./start-hbase.sh

一个基本配置的hbase分布式已经安装完毕
