修改系统文件句柄数和进程限制

发表于 2018-05-28 | 更新于 2018-05-29 | 分类于 linux

在生产环境运行hbase的时候，由于会同时使用很多文件，就会遇到 Can’t open so many files的问题。这个时候就需要设置ulimit和nproc的值

ulimit设置

我们可以用ulimit -a来查看所有限制值

[root@centos5 ~]# ulimit -a
core file size          (blocks, -c) 0
data seg size           (kbytes, -d) unlimited
scheduling priority             (-e) 0
file size               (blocks, -f) unlimited
pending signals                 (-i) 63454
max locked memory       (kbytes, -l) 64
max memory size         (kbytes, -m) unlimited
open files                      (-n) 1024
pipe size            (512 bytes, -p) 8
POSIX message queues     (bytes, -q) 819200
real-time priority              (-r) 0
stack size              (kbytes, -s) 8192
cpu time               (seconds, -t) unlimited
max user processes              (-u) 63454
virtual memory          (kbytes, -v) unlimited
file locks                      (-x) unlimited

其中 “open files (-n) 1024 “是Linux操作系统对一个进程打开的文件句柄数量的限制

临时做法：

ulimit -n 204800

永久做法，应该是修改/etc/security/limits.conf，添加上下面两句话

* soft nofile 65535
* hard nofile 65535

配置文件最前面的是指domain，设置为星号代表全局，另外你也可以针对不同的用户做出不同的限制。
这个当中的硬限制是实际的限制，而软限制，是warnning限制，只会做出warning

配置完成后退出用户登录后重新登陆后执行

ulimit -a

nproc设置

nproc的设置通过修改上边的/etc/security/limits.conf也行，但是20-nproc.conf里面有默认的配置会覆盖limits.conf里面的配置，这里需要注意以下。

系统是先加载limits.conf然后按照英文字母顺序加载limits.d目录下的配置文件，后加载配置覆盖之前的配置

通过/etc/security/limits.d/20-nproc.conf 修改

* soft nproc 204800  
* hard nproc 204800
