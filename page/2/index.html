发表于 2017-10-29 | 更新于 2018-05-24 | 分类于 linux

最近需要在现场的服务器上部署一个java项目，为了防止项目的停止，就在服务器上安装了一个monit，但是又怕停电了，还需要把monit设置为开机自启，流程看起来很简单，但是在实际的操作中确实遇到了各种各样的问题。特地记录下来，防止以后还会遇到这样的问题。另外有安装了一个teamviewer防止出现特殊情况。

前提条件

  • 系统 red hat enterprise linux 7.2
  • 软件monit-5.23.0
  • 软件teamviewer_12.0.85001.i686.rpm

安装步骤

修改yum源

由于 redhat的yum在线更新是收费的，如果没有注册的话不能使用，如果要使用，需将redhat的yum卸载后，重启安装，再配置其他源

  1. 卸载redhat自带的yum

    rpm -qa | grep yum | xargs rpm -e –nodeps

  • xargs是一条Unix和类Unix操作系统的常用命令。它的作用是将参数列表转换成小块分段传递给其他命令，以避免参数列表过长的问题
  • –nodeps 强制卸载，不管依赖性
  1. 下载yum需要的rpm包
    我这里使用的163的，网址http://mirrors.163.com/centos/7/os/x86_64/Packages/，
    在这个目录下找到需要的rpm包,下载后上传到服务器上

optional类

发表于 2017-10-17 | 更新于 2018-05-24 | 分类于 java基础

java8新增的另外一个实用的类就是optional了，此类主要是解决NullPointException的问题。但并不是简简单单的判断一个对象是否为空，而是应该换种思路来处理对象为空的问题。我们应该尽量不去使用isPresent()和get()方法，尽管官方解释中用到了这两个方法。

官方解释

A container object which may or may not contain a non-null value.If a value is present, {@code isPresent()} will return {@code true} and * {@code get()} will return the value.
翻译：这是一个可以为null的容器对象。如果值存在则isPresent()方法会返回true，调用get()方法会返回该对象。

这个解释可能会误导我们。我们真正应该使用的是下面这几个：

  1. public Optional map(Function<? super T, ? extends U> mapper)
  2. public T orElse(T other)
  3. public T orElseGet(Supplier<? extends T> other)
  4. public void ifPresent(Consumer<? super T> consumer)
  5. public Optional filter(Predicate<? super T> predicate)
  6. public Optional flatMap(Function<? super T, Optional> mapper)
  7. public T orElseThrow(Supplier<? extends X> exceptionSupplier) throws X

Stream语法

发表于 2017-10-16 | 更新于 2018-05-24 | 分类于 java基础

学习过lambda表达式后，肯定是要来学习一下它的好基友stream的。这两个的配合将是以后处理集合的主要方法。

1. 官方解释

A sequence of elements supporting sequential and parallel aggregate operations.
翻译：支持串行和并行聚合操作的一系列元素

2. 个人理解

其实就是更高级的Iterator.原始的Iterator只能一个一个的遍历元素并对其进行操作；Stream，用户只要给出需要对其包含的元素执行什么操作，比如“过滤掉长度大于10的字符串”、“获取每个字符串的首字母”等，而且还有并行操作的stream。在以往对数据进行聚合，要不就是依赖于关系型数据库，要不就是使用Iterator来遍历集合，完成相关的聚合，现在使用stream就能简单的实现。
