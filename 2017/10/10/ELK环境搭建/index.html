ELK环境搭建

Diposting di 2017-10-10 | Edited on 2018-05-23

一、单机版的安装

1. 上官网 下载软件包

2.1 elasticsearch安装

1. 解压安装包，创建elsearch用户组及elsearch用户

groupadd elsearch
useradd elsearch -g elsearch -p elasticsearch

2. 更改elasticsearch文件夹及内部文件的所属用户及组为elsearch:elsearch

chown-Relsearch:elsearchelasticsearch

elasticsearch为你elasticsearch的目录名称

3. 切换到elsearch用户再启动

su elsearch #切换账户cd elasticsearch/bin #进入你的elasticsearch目录下的bin目录 ./elasticsearch

4. 后台启动./elasticsearch -d

2.2 遇到的问题以及解决方案

1.ERROR: bootstrap checks failed
            max file descriptors [4096] for elasticsearch process likely too low, increase to at least [65536]
            max number of threads [1024] for user [lishang] likely too low, increase to at least [2048]

解决：切换到root用户，编辑limits.conf 添加类似如下内容

    vi /etc/security/limits.conf 
    添加如下内容:
    * soft nofile 65536
    * hard nofile 131072
    * soft nproc 2048
    * hard nproc 4096

2.问题：max virtual memory areas vm.max_map_count [65530] likely too low, increase to at least [262144]
解决：切换到root用户修改配置sysctl.conf
    vi /etc/sysctl.conf 
    添加下面配置：
    vm.max_map_count=655360
    并执行命令：
    sysctl -p

3. logstash安装

  1. 下载安装包
  2. 解压 tar -zxvf elasticsearch-5.6.1.tar.gz
  3. 运行bin/logstash -e’input { stdin { } } output { stdout {} }’
  4. 运行bin/logstash -e’input { stdin { } } output { stdout { codec => rubydebug } }’
  5. 添加配置文件config/logstash.conf

如果内存小的话，修改配置文件jvm.options中的-Xms和-Xmx

  1. 运行bin/logstash -f config/logstash.conf
  2. 后台运行nohup bin/logstash -f config/logstash.conf &

4. kibana安装

  1. 下载安装包
  2. 解压tar -zxvf kibana-5.6.1-linux-x86_64.tar.gz
  3. 修改config/kibana.yml
  4. 运行bin/kibana
  5. 后台运行 nohup bin/kibana &

5. filebeat安装

  1. 下载安装包
  2. 解压

  3. 修改配置文件filebeat.yml

  4. 启动
    nohup ./filebeat -e -c filebeat.yml &
