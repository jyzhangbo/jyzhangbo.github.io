opentsdb安装步骤

发表于 2017-11-06 | 更新于 2018-05-23

opentsdb安装步骤

环境
  • centos7.2
  • opentsdb2.3.0
jdk安装

此步骤请自行百度
大致步骤：

  1. 卸载centos自带的openjdk
  2. 下载tar包并解压
  3. 配置环境变量
  4. 使环境变量生效

    source /etc/profile

gnuplot安装

由于我就是没有安装gnuplot，导致的结果就是访问ip:4242的时候，查询数据一致报错

报错内容：

nice: gnuplot: No such file or directory

然后自己通过下载tar包的方式，安装了一下，但是还是报错：

谷歌了大半天，也没有解决了，这时候看到其他人的安装步骤，就决定重新试一下，结果好使了。

通过这三行命令就能成功安装了。

hbase安装

这里自己只是安装了一个单机版的，

  1. 下载tar包并解压
  2. 修改配置文件hbase-env.sh

    export HBASE_MANAGES_ZK=false

这里我只修改了这个，让hbase不启动自带的，而是使用我自己安装的zookeeper，但是只修改这个还不行，还需要下面的hbase.cluster.distributed配置

  1. 修改配置文件hbase-site.xml
  1. 启动

    ./start-hbase.sh

opentsdb安装
  1. 下载tar包，这里可能需要翻墙才能下载
  2. 进入目录编译

    ./build.sh

编译可能会出现错误：

Error 2 make[1]: Leaving directory/opt/app/opentsdb/opentsdb-2.3.0/build'
make: *** [all] Error 2

这个时候需要执行命令：

cp -r third_party ./build

然后再执行./build.sh

最后会生成build目录，在目录中会有脚本tsdb和一个jar包

  1. 建表

    env COMPRESSION=NONE HBASE_HOME=../hbase-0.94.27 src/create_table.sh

  2. 修改配置文件

vi src/opentsdb.conf

  1. 启动

    nohup build/tsdb tsd –config src/opentsdb.conf &

这个时候可以通过ip:4242查询数据

# opentsdb
